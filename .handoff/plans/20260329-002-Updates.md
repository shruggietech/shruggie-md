## Background

After the v0.3.2 release, the project owner discovered that clicking the native close button (the "X" in the Windows title bar) does not close the application. The window remains open with no visible error, freeze, or "Not Responding" state. The app is otherwise fully interactive.

### Root Cause

The `useWindowState` hook (`src/hooks/useWindowState.ts`) registers an `onCloseRequested` handler via the Tauri 2 window API. In Tauri 2, `onCloseRequested` works as follows:

1. The Rust runtime detects that a JavaScript listener exists for the `tauri://close-requested` event.
2. Instead of immediately destroying the window, it fires the event into the webview and waits for the JavaScript handler's Promise to resolve.
3. After the Promise resolves, if `event.preventDefault()` was not called, Tauri calls `window.destroy()`.

The current handler is `async` and performs a sequence of awaited IPC calls before the Promise resolves:

```
win.isMaximized()          â†’ IPC round-trip to Rust
win.outerSize()            â†’ IPC round-trip to Rust
win.outerPosition()        â†’ IPC round-trip to Rust
storage.setConfigValue()   â†’ tauri-plugin-sql SQLite write (Ă—4 or Ă—5)
```

If any one of these calls hangs (IPC channel contention during the shutdown sequence, SQLite connection teardown race, or plugin state inconsistency during the close lifecycle), the Promise never resolves. Tauri waits indefinitely, `destroy()` is never called, and the window silently stays open.

The `tauri.conf.json` window configuration is correct (no `closable: false`, decorations enabled, no Rust-side `on_window_event` handler intercepting the close). The issue is purely in the async JavaScript close handler.

### Fix Strategy

Two complementary changes:

1. **Timeout guard on the close handler.** Wrap the save logic in a `Promise.race` against a hard timeout (2 seconds). This guarantees the handler always resolves within a bounded time, regardless of IPC or SQLite behavior during shutdown. The window will close even if the save fails.

2. **Debounced save on move/resize events.** Persist window geometry incrementally (on move and resize, debounced to 1 second) so that the close handler is no longer the sole persistence path. If the close-time save races against shutdown and fails, the most recent geometry is already on disk from the last move/resize event. The close handler becomes a best-effort final write, not a critical bottleneck.

Together, these changes eliminate the hang and make window state persistence resilient to shutdown races.

## Phase 2 â€” Fix Implementation

### Task 2.1: Add Timeout Guard to Close Handler

In `src/hooks/useWindowState.ts`, refactor the `onCloseRequested` callback to wrap all save logic in a `Promise.race` with a hard timeout.

**Before (current broken pattern):**
```ts
const unlisten = await win.onCloseRequested(async () => {
  // Multiple awaited IPC and SQLite calls that can hang
  // ...
});
```

**After (timeout-guarded pattern):**
```ts
const unlisten = await win.onCloseRequested(async () => {
  const saveTimeout = new Promise<void>((resolve) =>
    setTimeout(resolve, 2000),
  );

  const saveState = (async () => {
    const s = storageRef.current;
    if (!s) return;

    try {
      const isMax = await win.isMaximized();
      if (!isMax) {
        const size = await win.outerSize();
        const pos = await win.outerPosition();
        await Promise.all([
          s.setConfigValue("window.width", JSON.stringify(size.width)),
          s.setConfigValue("window.height", JSON.stringify(size.height)),
          s.setConfigValue("window.x", JSON.stringify(pos.x)),
          s.setConfigValue("window.y", JSON.stringify(pos.y)),
        ]);
      }
      await s.setConfigValue("window.maximized", JSON.stringify(isMax));
    } catch {
      // Silent failure â€” don't block close
    }
  })();

  await Promise.race([saveState, saveTimeout]);
});
```

This guarantees the handler resolves within 2 seconds. If the save completes faster, the window closes immediately. If it hangs, the timeout fires and the window closes anyway (at the cost of potentially stale geometry, which Task 2.2 mitigates).

### Task 2.2: Add Debounced Save on Move/Resize Events

Add two new event listeners inside the same `init()` function in `useWindowState.ts`: one for `onMoved` and one for `onResized`. Both should save the current geometry to storage, debounced to 1000ms so rapid resize/drag sequences don't flood SQLite with writes.

**Implementation notes:**

- Use a single shared `setTimeout` handle for the debounce. Clear and reset it on each move or resize event.
- The debounced save function should perform the same geometry persistence logic as the close handler (read size/position, write to storage).
- Register both listeners after the initial restore logic, alongside the close handler.
- Store all unlisten functions and call them in the cleanup path.
- The move/resize save should also be wrapped in a try/catch to avoid unhandled rejections.
- Do NOT save geometry while the window is maximized (consistent with the close handler logic that only persists non-maximized dimensions).

**Pseudostructure:**
```ts
let debounceTimer: ReturnType<typeof setTimeout> | undefined;

const debouncedSave = () => {
  clearTimeout(debounceTimer);
  debounceTimer = setTimeout(async () => {
    const s = storageRef.current;
    if (!s) return;
    try {
      const isMax = await win.isMaximized();
      if (isMax) return; // Don't persist maximized dimensions
      const size = await win.outerSize();
      const pos = await win.outerPosition();
      await Promise.all([
        s.setConfigValue("window.width", JSON.stringify(size.width)),
        s.setConfigValue("window.height", JSON.stringify(size.height)),
        s.setConfigValue("window.x", JSON.stringify(pos.x)),
        s.setConfigValue("window.y", JSON.stringify(pos.y)),
      ]);
      await s.setConfigValue("window.maximized", "false");
    } catch {
      // Silent failure
    }
  }, 1000);
};

const unlistenMove = await win.onMoved(debouncedSave);
const unlistenResize = await win.onResized(debouncedSave);
```

Update the cleanup path to unlisten all three handlers (close, move, resize) and clear the debounce timer.
