## Phase 2 — Windows File Association Launch Fix

The app was not reading `argv[1]` at startup when launched via Windows shell "Open With" verb. A new `useCliArgs` hook was implemented to:

1. Read launch arguments from Tauri's CLI plugin.
2. Resolve the file path before the welcome-document/last-session restoration logic runs.
3. Fall back to normal startup flow when the path is absent, non-existent, or unreadable.

Test coverage: `use-cli-args.test.ts` — 3 tests.

---

## Phase 3 — Markdown Rendering Fidelity

### Task 3.1: Bullet Glyph Restoration

A global CSS reset (Tailwind preflight) was stripping `list-style` from all list elements. The preview stylesheet now explicitly restores `disc/circle/square` for `ul` nesting levels and `decimal/lower-alpha/lower-roman` for `ol` nesting levels.

### Task 3.2: Inline Code Color

A new design token `--color-inline-code` was defined (muted green) and applied to `code:not(pre code)` elements in both light and dark themes.

### Task 3.3: GitHub-Aligned Preview Style

`src/styles/preview.css` was updated to match conventional GitHub markdown rendering conventions: heading scale, `h1`/`h2` bottom borders, blockquote styling, table zebra striping, horizontal rule height, and paragraph spacing.

### Task 3.4: Markdown Engine Dropdown Audit

**Outcome: Classification 1** — All three engines are functionally wired and produce near-identical output on common markdown.

Technical findings (migrated from sprint plan notes):

- Routing status: Wired end-to-end.
- Routing detail: Settings dropdown writes `config.engine.activeEngine`.
- Routing detail: `App` passes selected engine into `Preview` and export hooks.
- Routing detail: `renderMarkdown(source, engineId)` dispatches to the mapped engine implementation.
- Installed engines: `markdown-it`, `marked`, `remark` (via `remark` + `remark-gfm` + `remark-html`).
- Verified difference surfaces:
  - Typographer input (`"quotes" -- dash`) emits typographic quotes/en-dash in markdown-it; marked/remark do not.
  - Footnote output differs in generated HTML structure/attributes across all three engines; marked behavior differs most without explicit footnote extension.
  - Task list HTML classes/attributes differ between engines while visual rendering remains close under shared preview CSS.

## Phase 4 — Refresh Button and Unsaved-Changes Guard

A Refresh toolbar button was added to the Actions zone. Clicking Refresh with a clean document re-reads the file from disk and re-renders. With unsaved changes, a modal dialog warns before discarding edits.

The modal follows the existing pattern from the About modal (v0.3.6).

Test coverage: `refresh-flow.test.tsx` — 4 tests.

---

## Phase 5 — Font Family Setting Dropdown

The free-text font input was replaced with a curated `<select>` dropdown containing 12 named options. The default is "System Default". Previously persisted values not matching a curated entry are silently migrated to System Default.

Font list module: `src/config/previewFonts.ts`.

Test coverage: `preview-fonts.test.tsx` — 17 tests.
