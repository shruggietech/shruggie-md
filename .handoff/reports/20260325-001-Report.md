## 5. Architecture As-Built

### 5.1 Runtime Platform Detection

```
getPlatform() in src/platform/platform.ts
  │
  ├── window.__TAURI__ or __TAURI_INTERNALS__ present?
  │   └── YES → TauriAdapter (filesystem, watcher, dialogs, CLI)
  │
  └── NO → PwaAdapter (IndexedDB config, no filesystem)
       │
       (Chrome extension context uses ChromeAdapter directly via
        its own entry points — content.ts and popup.ts — not
        through the React app's getPlatform() path)
```

### 5.2 Data Flow

```
User opens file → PlatformAdapter.openFileDialog()
                → PlatformAdapter.readFile(path)
                → App state: { content, fileName, filePath }
                → Preview: renderMarkdown(content, engineId)
                           → engine.compile(source)
                           → sanitizeHtml(html)
                           → set innerHTML
                           → highlightCodeBlocks(container)

User edits → Editor onChange (CodeMirror updateListener)
           → 150ms debounce
           → App state: { content }
           → Preview re-renders

User saves → PlatformAdapter.writeFile(path, content)
             OR trigger browser download (non-desktop)

Config changes → ConfigProvider.updateConfig(section, values)
              → PlatformAdapter.writeConfig(fullConfig)
              → useConfigEffects applies CSS custom property updates
              → Components re-render with new config values
```

### 5.3 Build Outputs

| Target | Entry Point | Build Command | Output |
|--------|------------|---------------|--------|
| Web/PWA | `index.html` → `src/main.tsx` | `pnpm run build` | `dist/` (app + service worker) |
| Desktop | Tauri wraps `dist/` | `pnpm tauri:build` | MSI/DMG/AppImage |
| Chrome Extension | `extension/content.ts` + `popup.ts` | `pnpm run build:extension` | `extension/dist/` |

## 9. Incomplete Work & Stubs

### 9.1 Linter Adapters (Priority: Medium)

**Files:** `src/linters/markdownlint-adapter.ts`, `src/linters/remark-lint-adapter.ts`

Both files implement the `LinterAdapter` interface but return empty diagnostic arrays. The full implementation requires:

1. Install linter packages:
   ```bash
   pnpm add markdownlint markdownlint-rule-helpers
   pnpm add remark-lint remark-preset-lint-recommended unified
   ```

2. Implement `markdownlint-adapter.ts`:
   - Import markdownlint
   - Run against source string
   - Map markdownlint results to `LintDiagnostic[]` (convert line/column to character offsets)

3. Implement `remark-lint-adapter.ts`:
   - Import remark + remark-lint + preset
   - Process source through remark pipeline

## 13. Recommendations

### 13.1 Immediate (Before Release)

1. **Commit and tag.** The entire MVP codebase is untracked. Create a single comprehensive commit (or a series of sprint-based commits if auditability is preferred) and tag `v0.1.0`.

2. **Implement linter adapters.** This is the single largest functional gap. The plumbing (config toggle, Settings UI, CodeMirror lint gutter, adapter interface) is all in place — only the `lint()` method bodies are empty. This is a clean, well-scoped task for one AI agent session.

3. **Tighten CSP.** The null CSP in `tauri.conf.json` is a security risk for production. Add a restrictive policy.

4. **Test on real platforms.** The automated tests cover logic but not platform integration. At minimum, verify:
   - Desktop: file open, save, live reload, library mount
   - Chrome: content script activation on a raw .md URL
   - PWA: offline mode after install

### 13.2 Near-Term (Post-Release)

1. **Code splitting refinement.** The CodeMirror vendor chunk is 1.6 MB because `@codemirror/language-data` includes every language grammar. A curated list of 20–30 common languages would reduce this by ~70%.

2. **Atomic file save.** The Tauri adapter uses direct `writeTextFile()`. The spec calls for write-to-temp-then-rename. This can be implemented in the Rust backend for true atomicity.

3. **HTML export save dialog.** On desktop, HTML export currently triggers a download rather than a native save-as dialog. The platform adapter's `openFileDialog` could be extended with a save variant.

4. **Remove unused `tauri-plugin-cli` dependency** from Cargo.toml to reduce binary size.
