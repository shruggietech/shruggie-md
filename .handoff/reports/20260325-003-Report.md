## 1. Executive Summary

The Windows application launch failure has been diagnosed and fixed. The root cause was an **invalid `plugins.fs` configuration** in `tauri.conf.json` that caused a deserialization panic at startup, preventing the Tauri window from ever opening. Additionally, the Windows install path, NSIS configuration, Cargo.toml metadata, and spec §10.5 have all been corrected.

### Key Outcome

`pnpm tauri dev` now compiles and launches the application window successfully on Windows. All 258 tests pass, zero TypeScript errors.

## 3. Root Cause Analysis

### Primary Cause: Invalid `plugins.fs.scope` Configuration

The `tauri.conf.json` contained:

```json
"plugins": {
  "fs": {
    "scope": {
      "allow": ["**"]
    }
  }
}
```

In Tauri 2, the `tauri-plugin-fs` configuration schema does **not** accept a `scope` field. The only accepted field is `requireLiteralLeadingDot`. File system scoping in Tauri 2 is handled exclusively through the capabilities system (`src-tauri/capabilities/default.json`), which was already correctly configured with granular `fs:allow-*` permissions.

This invalid config caused the Tauri runtime to panic during plugin initialization with:
```
Error deserializing 'plugins.fs' within your Tauri configuration: unknown field `scope`, expected `requireLiteralLeadingDot`
```

The panic occurred before window creation, making the failure appear as "application doesn't launch" with no visible UI.

### Why This Wasn't Caught

1. The Vite frontend builds and serves independently — `pnpm build` and `pnpm run dev` succeed without Tauri.
2. The Rust code compiles cleanly — `cargo build` and `cargo check` pass because the config error is a runtime deserialization failure, not a compile-time error.
3. CI verified compilation and test passage, not actual application launch.
4. The `scope` field was valid in Tauri v1's fs plugin but was removed in v2's different security model.

## 4. Changes Made

### `src-tauri/tauri.conf.json`

1. **Fixed `plugins.fs` config**: Replaced invalid `scope` block with `{ "requireLiteralLeadingDot": false }`.
2. **Changed `identifier`**: `com.shruggietech.shruggie-md` → `com.shruggie-tech.shruggie-md` (for correct install path derivation).
3. **Added `bundle.windows` block**:
   - `nsis.installMode`: `"currentUser"` — installs to `%LOCALAPPDATA%`
   - `nsis.installerIcon`: `"icons/icon.ico"`
   - `nsis.displayLanguageSelector`: `false`
   - `webviewInstallMode`: `{ "type": "downloadBootstrapper", "silent": true }` — ensures WebView2 is available

### `src-tauri/Cargo.toml`

1. **Fixed authors**: `["you"]` → `["William Thompson"]`
2. **Removed unstable feature**: `features = ["unstable"]` → `features = []`

### `src-tauri/src/lib.rs`

1. **Removed unused import**: `use tauri::Manager;` deleted

### `shruggie-md-spec.md`

1. **Updated §10.5 Windows path**: `%LOCALAPPDATA%\shruggie-md\` → `%LOCALAPPDATA%\shruggie-tech\shruggie-md\`
